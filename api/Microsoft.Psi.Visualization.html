    ObservableDataCollection<T>

    The purpose of ObservableDataCollection is to optimize exposing non-bindable lists to XAML binding. It does this in two ways:

    1. It surfaces as an ObservableCollection which minimizes item removals and resets.
    2. It wraps non IPropertyChangeNotify objects so that they can be updated wholesale (via indirection in a ObservableDataItem) so that updates don't cause a collection change to force bindings to update.

    ObservableKeyedCache<TKey, TItem>

    Represents a dynamic data cache whose keys are embedded in the values and provides notifications when items get added, removed, or when the whole list is refreshed.

    The cache supports views onto the data. These views support changes to the underlying data and being updated by the underlying data.

    In future versions the underlying data will be to be released if there is no view present for a given data range.
